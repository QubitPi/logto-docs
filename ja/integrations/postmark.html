Postmark メールサービスを使用してメール認証を設定する

Logto の Postmark メールサービス用コネクター。

ヒント:

このガイドでは、Logto コネクターに関する基本的な知識を持っていることを前提としています。もし持っていない場合は、コネクターの設定 ガイドを参照して始めてください。

はじめに​

Postmark は、トランザクションおよびマーケティングメール用のメールプラットフォームです。そのメール送信機能を使用して、認証コード を送信できます。

Postmark アカウントを登録する​

Postmark のウェブサイトで新しいアカウントを作成します。すでにアカウントをお持ちの場合は、このステップをスキップできます。

コネクターを設定する​

Postmark のサーバー設定で見つけたサーバートークンを serverToken フィールドに入力します。

送信者の From アドレス を fromEmail フィールドに入力します。

完全なユーザーフローを有効にするには、Register、SignIn、ForgotPassword、Generic の usageType を持つテンプレートが必要です。

以下は、Postmark コネクターテンプレートの JSON の例です。

[
  {
    "usageType": "Register",
    "templateAlias": "logto-register",
  },
  {
    "usageType": "SignIn",
    "templateAlias": "logto-sign-in",
  },
  {
    "usageType": "ForgotPassword",
    "templateAlias": "logto-forgot-password",
  },
  {
    "usageType": "Generic",
    "templateAlias": "logto-generic",
  },
]

Postmark メールコネクターをテストする​

メールアドレスを入力し、「送信」をクリックして設定が機能するかどうかを「保存して完了」する前に確認できます。

以上です。サインイン体験でコネクターを有効にする のを忘れないでください。

設定タイプ​

名前タイプ
serverTokenstring
fromEmailstring
テンプレートプロパティタイプ列挙値
usageTypeenum string'Register' | 'SignIn' | 'ForgotPassword' | 'Generic'
templateAliasstringN/A
