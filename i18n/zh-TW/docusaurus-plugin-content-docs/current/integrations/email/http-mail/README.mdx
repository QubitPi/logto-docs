# 使用 HTTP 呼叫設定電子郵件驗證

Logto 的官方 HTTP 電子郵件連接器。

<GuideTip />

## 開始使用 \{#get-started}

HTTP 電子郵件連接器允許你通過 HTTP 呼叫發送電子郵件。要使用 HTTP 電子郵件連接器，你需要擁有自己的電子郵件服務，該服務提供用於發送電子郵件的 HTTP API。當 Logto 需要發送電子郵件時，將調用此 API。例如，當使用者註冊時，Logto 將調用 HTTP API 發送驗證電子郵件。

## 設定 HTTP 電子郵件連接器 \{#set-up-http-email-connector}

要使用 HTTP 電子郵件連接器，你需要設置一個 Logto 可以調用的 HTTP 端點，以及一個可選的端點授權權杖。

## 載荷 \{#payload}

當需要發送電子郵件時，HTTP 電子郵件連接器將以下載荷發送到端點：

```json
{
  "to": "foo@logto.io",
  "type": "SignIn",
  "payload": {
    "code": "123456"
  }
}
```

你可以在 [電子郵件範本](/connectors/email-connectors/built-in-email-service/#unified-email-templates) 中找到所有類型，以及在 [connector-kit](https://github.com/logto-io/logto/tree/master/packages/toolkit/connector-kit/src/types/passwordless.ts) 中找到 `SendMessageData` 的完整類型定義。
