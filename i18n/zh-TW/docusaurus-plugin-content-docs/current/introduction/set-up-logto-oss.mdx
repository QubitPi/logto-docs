# 設定 Logto OSS

本節涵蓋基本設定步驟和關鍵操作，以有效地交付產品並管理你的開發工作流程，適用於 [Logto 開源服務 (OSS)](https://github.com/logto-io/logto)。

## 開始使用 Logto OSS \{#get-started-with-logto-oss}

按照 [入門指南](/logto-oss/get-started-with-oss/) 立即啟動 Logto，之後你可以參考完整的 [部署指南](/logto-oss/deployment-and-configuration) 以用於生產環境。

## Logto OSS 支援的功能 \{#feature-supported-by-logto-oss}

Logto OSS 支援 Logto 服務的大多數核心功能，並定期更新和維護。

一些高級功能目前僅限於 Logto Cloud 版本，包括 [Logto Protected App](/integrate-logto/protected-app)、[免費電子郵件服務](/connectors/email-connectors/built-in-email-service)、[IdP 發起的單一登入 (SSO)](/end-user-flows/enterprise-sso/idp-initiated-sso) 和 [SAML IdP](/integrate-logto/third-party-applications)。

## 保持 Logto 版本更新 \{#stay-updated-with-logto-releases}

為了讓你的 Logto 實例保持最新功能，請務必關注 [Logto GitHub Releases](https://github.com/logto-io/logto/releases) 頁面，你可以在那裡找到所有的版本日誌。

閱讀 [升級指南](/logto-oss/upgrading-oss-version) 以了解如何在不更改代碼或資料庫結構的情況下升級 Logto。
