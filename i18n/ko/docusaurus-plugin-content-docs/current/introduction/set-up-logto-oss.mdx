# Logto OSS 설정

이 섹션에서는 [Logto 오픈 소스 서비스 (OSS)](https://github.com/logto-io/logto)를 효과적으로 제공하고 개발 워크플로를 관리하기 위한 기본 설정 단계와 주요 작업을 다룹니다.

## Logto OSS 시작하기 \{#get-started-with-logto-oss}

[시작 가이드](/logto-oss/get-started-with-oss/)를 따라 오늘 Logto를 시작하세요. 이후에는 프로덕션 사용을 위한 전체 [배포 가이드](/logto-oss/deployment-and-configuration)를 참조할 수 있습니다.

## Logto OSS가 지원하는 기능 \{#feature-supported-by-logto-oss}

Logto OSS는 Logto 서비스의 대부분의 핵심 기능을 지원하며 정기적으로 업데이트되고 유지 관리됩니다.

일부 고급 기능은 현재 Logto Cloud 버전에만 독점적으로 제공되며, 여기에는 [Logto Protected App](/integrate-logto/protected-app), [무료 이메일 서비스](/connectors/email-connectors/built-in-email-service), [IdP-initiated SSO](/end-user-flows/enterprise-sso/idp-initiated-sso), 및 [SAML IdP](/integrate-logto/third-party-applications)가 포함됩니다.

## Logto 릴리스 최신 상태 유지 \{#stay-updated-with-logto-releases}

최신 기능으로 Logto 인스턴스를 최신 상태로 유지하려면 [Logto GitHub Releases](https://github.com/logto-io/logto/releases) 페이지를 팔로우하세요. 모든 릴리스 로그를 찾을 수 있습니다.

코드나 데이터베이스 스키마를 변경하지 않고 Logto를 업그레이드하는 방법을 배우려면 [업그레이드 가이드](/logto-oss/upgrading-oss-version)를 읽어보세요.

## Logto OSS에 기여하기 \{#contributing-to-logto-oss}

Logto에 기여해 주셔서 감사합니다! 여기에 [기여 지침](/logto-oss/contribution)이 있습니다.
