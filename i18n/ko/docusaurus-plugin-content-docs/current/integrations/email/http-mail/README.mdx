# HTTP 호출을 통한 이메일 인증 설정

HTTP 이메일을 위한 공식 Logto 커넥터입니다.

<GuideTip />

## 시작하기 \{#get-started}

HTTP 이메일 커넥터를 사용하면 HTTP 호출을 통해 이메일을 보낼 수 있습니다. HTTP 이메일 커넥터를 사용하려면 이메일을 보내기 위한 HTTP API를 노출하는 자체 이메일 서비스가 필요합니다. Logto는 이메일을 보낼 필요가 있을 때 이 API를 호출합니다. 예를 들어, 사용자가 등록할 때 Logto는 인증 이메일을 보내기 위해 HTTP API를 호출합니다.

## HTTP 이메일 커넥터 설정 \{#set-up-http-email-connector}

HTTP 이메일 커넥터를 사용하려면 Logto가 호출할 수 있는 HTTP 엔드포인트를 설정해야 합니다. 그리고 엔드포인트에 대한 선택적 인가 토큰이 필요합니다.

## 페이로드 \{#payload}

HTTP 이메일 커넥터는 이메일을 보낼 필요가 있을 때 다음과 같은 페이로드를 엔드포인트로 전송합니다:

```json
{
  "to": "foo@logto.io",
  "type": "SignIn",
  "payload": {
    "code": "123456"
  }
}
```

모든 유형은 [이메일 템플릿](/connectors/email-connectors/built-in-email-service/#unified-email-templates)에서 확인할 수 있으며, `SendMessageData`의 전체 유형 정의는 [connector-kit](https://github.com/logto-io/logto/tree/master/packages/toolkit/connector-kit/src/types/passwordless.ts)에서 확인할 수 있습니다.
