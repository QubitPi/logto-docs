Logto는 OpenID Connect (OIDC) 인증 (Authentication)과 OAuth 2.0 인가 (Authorization)를 기반으로 합니다. 이는 여러 애플리케이션에 걸쳐 연합 아이덴티티 관리를 지원하며, 일반적으로 싱글 사인온 (SSO)이라고 불립니다.

{props.type} 애플리케이션을 생성하려면, 다음 단계를 따르세요:

1. [Logto Console](https://cloud.logto.io) 을 엽니다. "Get started" 섹션에서 "View all" 링크를 클릭하여 애플리케이션 프레임워크 목록을 엽니다. 또는 [Logto Console > Applications](https://cloud.logto.io/to/applications) 로 이동하여 "Create application" 버튼을 클릭할 수 있습니다.
   <img src={getStartedImage} alt="Get started" />
2. 열리는 모달에서 "{props.type}" 섹션을 클릭하거나 왼쪽의 빠른 필터 체크박스를 사용하여 사용 가능한 모든 "{props.type}" 프레임워크를 필터링합니다. "{props.framework}" 프레임워크 카드를 클릭하여 애플리케이션 생성을 시작하세요.
   <img src={frameworkListImage} alt="Frameworks" />
3. 애플리케이션 이름을 입력합니다. 예를 들어, "Bookstore"를 입력하고 "Create application"을 클릭합니다.

🎉 짜잔! Logto에서 첫 번째 애플리케이션을 생성했습니다. 자세한 통합 가이드가 포함된 축하 페이지를 보게 될 것입니다. 가이드를 따라 애플리케이션에서의 경험이 어떻게 될지 확인하세요.
