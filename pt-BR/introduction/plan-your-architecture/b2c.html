Arquitetura B2C

Arquitetura​

Em aplicativos de consumo, os usuários possuem totalmente suas contas e controlam sua identidade e autenticação, sem envolvimento de outras partes "intermediárias" como empresas ou organizações. Esta é a principal distinção entre arquiteturas de identidade B2C e B2B.

Construa seu sistema de identidade B2C​

Vários métodos de autenticação: senha, sem senha e login social.

App autenticador, Passkey, Códigos de backup. Permita que os usuários ativem MFA de forma independente.

Use Logto Account API para desenvolver configurações de conta e páginas de perfil de usuário para seus usuários finais.

Defina papéis e atribua permissões correspondentes para controlar as ações dos usuários.

Gerencie identidades de usuários no Logto Console ou Logto Management API.

Permita que os usuários autorizem aplicativos de terceiros por conta própria.

Conecte seus requisitos ao kit de suporte do Logto​

Esta arquitetura inclui duas partes principais envolvidas no cenário de gerenciamento. Dependendo de suas necessidades e objetivos específicos, todas ou apenas algumas dessas partes podem estar envolvidas.

Resumimos casos de uso comuns, destacando os principais objetivos de cada tarefa de gerenciamento de identidade do usuário e os produtos e APIs relacionados que oferecemos. Você pode mapear suas necessidades para nossos serviços para começar rapidamente.

UsuáriosObjetivoProdutos e APIs do Logto
DesenvolvedoresGerenciar e proteger o sistema de identidade do usuário e trabalhar diretamente com o banco de dados de identidade.
  • Logto Console
  • Logto Management API
Usuário final/ConsumidoresGerenciar sua própria autenticação e informações pessoais.
  • Logto Management API
  • Account API
