Arquitectura multi-tenant para servicios B2B

Arquitectura​

Las aplicaciones B2B suelen utilizar una arquitectura multi-tenant. En estas aplicaciones, los usuarios poseen sus cuentas y gestionan su identidad y autenticación, con la participación de otras partes como empresas u organizaciones. Las identidades de los usuarios finales a menudo no son consumidores individuales, sino empleados o colaboradores dentro de una organización empresarial.

Características B2B​

La función de Organizaciones es ideal para aplicaciones SaaS multi-tenant y B2B, apoyando a equipos, organizaciones y empresas como clientes.

Logto admite SSO iniciado por SP con SAML y OIDC, conectándose a proveedores como Okta, Azure AD y Google Workspace.

Características genéricas​

Varios métodos de autenticación: contraseña, sin contraseña e inicio de sesión social.

Aplicación de autenticación, Passkey, Códigos de respaldo. Permite a los usuarios habilitar MFA de manera independiente.

Usa Logto Account API para desarrollar configuraciones de cuenta y páginas de perfil de usuario para tus usuarios finales.

Define roles y asigna permisos correspondientes para controlar las acciones de los usuarios.

Gestiona identidades de usuario en Logto Console o Logto Management API.

Permite a los usuarios autorizar aplicaciones de terceros por sí mismos.

Conecta tus requisitos con el kit de herramientas de soporte de Logto​

Esta arquitectura incluye tres partes principales involucradas en el escenario de gestión. Dependiendo de tus necesidades y objetivos específicos, todas o solo algunas de estas partes pueden estar involucradas.

Hemos resumido casos de uso comunes, destacando los objetivos clave de cada usuario en la gestión de tareas y los productos y APIs relacionados que ofrecemos. Puedes mapear tus necesidades a nuestros servicios para comenzar rápidamente.

UsuariosObjetivoProductos y APIs de Logto
DesarrolladoresGestionar y proteger el conjunto de identidades de usuarios y trabajar directamente con la base de datos de identidades.
  • Logto Console
  • Logto Management API
Administrador de la organizaciónGestión de la organización de autoservicio con características personalizadas, dando a los administradores de la organización control total sobre la gestión de identidades.

  1. Crear organizaciones por su cuenta
  2. Invitar a miembros a unirse
  3. Agregar o eliminar información de miembros y actualizar sus roles
  • Logto Management API
Usuario final / Miembro de la organizaciónGestionar su propia autenticación a nivel de sistema e información personal.
  • Logto Management API
  • Account API
