终端用户流程

终端用户流程涵盖了所有用户交互的验证过程，分为以下几类：

  • 认证 (Authentication) 流程：涉及用户登录、注册和重置密码。Logto 在 控制台 > 登录体验 中提供了开箱即用的体验，你也可以通过 Experience API 自定义你的 UI。
  • 账户流程：涉及账户设置、用户资料和安全验证。可通过 Account API 或 Management API 实现。
  • 组织 (Organization) 流程：适用于多租户产品，如 B2B 或 SaaS 服务。涉及组织创建、成员邀请和管理。可通过 Management API 集成到你的应用中。

认证 (Authentication) 流程​

流程详情
注册与登录
  • 邮箱 / 手机 / 用户名注册
  • 邮箱 / 手机 / 用户名登录
  • 社交登录
  • 重置密码
  • 条款与隐私
  • 禁用用户注册
企业单点登录 (SSO)
  • SP 发起的 SSO
  • IdP 发起的 SSO
多因素认证 (MFA)
  • 认证器应用 OTP
  • Passkeys (WebAuthn)
  • 备份码
认证 (Authentication) 参数
  • 首屏
  • 直接登录
魔法链接（一次性令牌）
  • 组织成员邀请
  • 禁用注册时用户邀请
  • 使用魔法链接登录或注册
授权第三方应用
  • OIDC / OAuth 应用的用户授权页面 (Consent screen)
登出
  • 在客户端清除令牌和本地会话
  • 在 Logto 清除登录会话
  • 联合登出：后端通道登出

本节介绍了 Logto 的预构建 UI，帮助你快速实现流畅的登录体验，加速产品上线。如果你需要更灵活地自定义登录 UI，可以尝试 Logto Experience API 的“自定义你的 UI”功能。
