Hugging Face 소셜 로그인을 설정하세요

Hugging Face 소셜 로그인에 대한 공식 Logto 커넥터입니다.

팁:

이 가이드는 Logto 커넥터에 대한 기본적인 이해를 전제로 합니다. 익숙하지 않은 분들은 커넥터 가이드를 참조하여 시작하세요.

시작하기​

Hugging Face 커넥터를 통해 최종 사용자는 Hugging Face OAuth / OpenID Connect 흐름을 통해 자신의 Hugging Face 계정을 사용하여 애플리케이션에 로그인할 수 있습니다.

Hugging Face 계정으로 로그인하기​

Hugging Face 웹사이트로 이동하여 Hugging Face 계정으로 로그인하세요. 계정이 없으면 새 계정을 등록할 수 있습니다.

Hugging Face에서 OAuth 앱 생성하기​

OAuth 앱 생성 가이드를 따라 새 애플리케이션을 등록하세요.

생성 과정에서 다음 정보를 제공해야 합니다:

  • 애플리케이션 이름: 애플리케이션의 이름.
  • 홈페이지 URL: 애플리케이션의 홈페이지 또는 랜딩 페이지 URL.
  • 로고 URL: 애플리케이션의 로고 URL.
  • 스코프: OAuth 앱에 허용된 스코프. Hugging Face 커넥터의 경우, 사용자의 프로필 정보를 얻기 위해 profile을, 사용자의 이메일 주소를 얻기 위해 email을 주로 사용합니다. 이러한 스코프를 사용하려면 Hugging Face OAuth 앱에서 허용되었는지 확인하세요.
  • 리디렉션 URI: 사용자가 인증된 후 리디렉션할 URL. Hugging Face 커넥터를 생성할 때 Logto 관리 콘솔에서 또는 생성된 Hugging Face 커넥터 세부 정보 페이지에서 리디렉션 URI를 찾을 수 있습니다.

Hugging Face OAuth 앱 관리하기​

연결된 애플리케이션 페이지로 이동하여 기존 OAuth 앱을 추가, 편집 또는 삭제할 수 있습니다. 해당 OAuth 앱 설정 페이지에서 Client ID를 찾고 App secrets를 생성할 수 있습니다.

커넥터 구성하기​

이전 섹션에서 언급한 OAuth 앱 세부 정보 페이지에서 얻은 Client ID와 App Secret으로 clientId와 clientSecret 필드를 작성하세요.

scope는 Hugging Face 지원 스코프의 공백으로 구분된 목록입니다. 제공되지 않으면 기본적으로 profile로 설정됩니다. Hugging Face 커넥터의 경우, 사용하고자 하는 스코프는 profile과 email입니다. profile 스코프는 사용자의 프로필 정보를 얻기 위해 필요하며, email 스코프는 사용자의 이메일 주소를 얻기 위해 필요합니다. Hugging Face OAuth 앱에서 이러한 스코프가 허용되었는지 확인하세요 ( Hugging Face에서 OAuth 앱 생성하기 섹션에서 구성됨).

구성 유형​

이름유형
clientIdstring
clientSecretstring
scopestring

Hugging Face 커넥터 테스트하기​

이제 Hugging Face 커넥터가 사용 가능해야 합니다. 로그인 경험에서 소셜 커넥터 활성화를 잊지 마세요.
